Project Report: Leaderboard Challenge
Approach
I built this application using a design-first "vibe coding" approach. As a motion designer, I focused on translating visual hierarchy into technical prompts, using AI to handle the manual coding process.

Tools Used
Claude 3.5 Sonnet & Opus (Primary Development)

Gemini (Data Generation and Technical Descriptions)

Figma (Layout and Structural Analysis)

Tailwind CSS

Data Handling
To comply with the challenge requirements, all original data was replaced with a fictional dataset generated by Gemini. I replaced real names, titles, and departments with a space-themed system (using planets and fictional units) to ensure 0% overlap with the original data.

Process
Analysis: I structured the page components in Figma to define the layout hierarchy.

Technical Prompting: I used Gemini to describe each UI block in technical terms based on clean screenshots.

Development: I used these descriptions and screenshots to prompt Claude. While I started with Haiku, I moved to Sonnet and Opus for better handling of complex CSS and mobile responsiveness.

Refinement: I adjusted the UI by describing visual discrepancies in plain language and using Gemini to help translate those into specific CSS/HTML fixes for Claude.
